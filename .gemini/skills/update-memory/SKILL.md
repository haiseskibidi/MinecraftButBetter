# Update Memory Workflow

Этот навык обеспечивает поддержание Single Source of Truth (SSOT) проекта MinecraftButBetter.

## Когда использовать
- После завершения реализации новой фичи.
- После исправления бага.
- После рефакторинга кода (обновление `code_map.md`).
- После изменения игровых механик или лора.

## Процедура обновления

### 1. Текущее состояние (current_state.md)
- Перемести выполненные задачи из "В работе" в "Реализованные фичи" (с кратким описанием).
- Добавь новые задачи в раздел "В работе".
- Отрази прогресс по Milestone в разделе Roadmap.

### 2. Архитектура (architecture.md)
- Отрази новые зависимости между пакетами.
- Добавь описание новых систем (например, "Система Энергии").
- Обнови схемы взаимодействия, если они изменились.

### 3. Карта кода (code_map.md)
- Если были добавлены новые классы или изменены сигнатуры важных методов, обнови соответствующий раздел.
- Сохраняй формат: Путь к файлу, Назначение, Функции, Зависимости.

### 4. Лор и механики (lore.md)
- Уточняй параметры механик (например, значения уровней шума).
- Добавляй новые детали сюжета.

## Принципы
- **Компактность:** Не пиши "воду", только технические факты.
- **Хирургические правки:** При обновлении файлов используй точечные изменения (replace). Избегай полной перезаписи (write_file) существующих md-файлов, чтобы сохранить историю и структуру.
- **Своевременность:** Обновляй файлы СРАЗУ после выполнения основного действия.
- **Точность:** Любое утверждение в памяти должно соответствовать реальному коду.
