# План: Динамический Хват (Dynamic Grip System)

## Цель
Реализовать гибкую, Data-Driven систему сгибания пальцев при удержании предметов. Пальцы должны плавно облегать форму предмета (например, плоский хват для листа металла, сжатый кулак для рукояти кирки, щипковый хват для мелких предметов). Никакого хардкода в Java, все углы сгиба должны задаваться в JSON предмета.

## Архитектура решения

Мы расширим компонент `viewmodel` в JSON-файлах предметов, добавив секцию `grip`. Эта секция будет содержать целевые углы поворота для костей пальцев.

### 1. Изменения в Data-слое (JSON)
Добавление секции `grip` в `viewmodel`:
```json
"viewmodel": {
  "socket": "socket_palm",
  "translation": [0, 0, 0],
  "rotation": [0, 90, 0],
  "scale": 0.5,
  "grip": {
    "thumb": [0, -20, -40],
    "index": [45, 0, 0],
    "fingers": [60, 0, 0]
  }
}
```
*Углы задаются в градусах: [Pitch (X), Yaw (Y), Roll (Z)].*

### 2. Изменения в Java (Рекорды и Парсинг)
*   **`ViewmodelComponent.java`**: Обновить рекорд, добавив поле `GripComponent grip`.
*   **`GripComponent.java`**: Новый рекорд, хранящий `float[] thumb`, `float[] index`, `float[] fingers`.
*   **`DataLoader.java`**: Добавить логику чтения поля `grip` при парсинге `ViewmodelComponent`. Если поля нет, используются дефолтные нулевые (или слегка расслабленные) углы.

### 3. Изменения в Физике (Анимация хвата)
*   **`Player.java` (или `ViewmodelController`)**: В цикле обновления анимаций проверять предмет в руке.
*   Извлекать целевые углы из `grip`.
*   Плавно интерполировать (`lerp`) текущий поворот костей `thumb_r/l`, `index_r/l`, `fingers_r/l` к целевым углам.
*   Если рука пуста, плавно возвращать пальцы в базовое "расслабленное" состояние (например, легкий изгиб на 10-15 градусов).

### 4. Тестирование и Настройка
*   Настроить хват для `magnetite` (мелкий предмет, пальцы сжаты: index ~60, fingers ~70).
*   Настроить хват для `metal_sheet` (плоский предмет, пальцы почти прямые: index ~10, fingers ~10, thumb прижат с другой стороны).

## Преимущества
*   **Гибкость**: Для любого нового предмета можно задать уникальный хват просто через JSON.
*   **Плавность**: Использование `lerp` обеспечит красивое сжимание и разжимание кисти при смене предметов.
*   **Отсутствие хардкода**: Никаких "if (item == magnetite)" в коде рендеринга.
