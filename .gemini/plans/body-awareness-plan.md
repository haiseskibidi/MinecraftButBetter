# План реализации Body Awareness (Ощущение тела)

Этот документ описывает проектирование и внедрение системы отображения тела персонажа от первого лица в проекте **Zenith**.

## 1. Цели системы
- Углубление погружения (Immersive First Person).
- Визуализация движений персонажа при паркуре (ноги упираются в стены, болтаются при висении).
- Процедурная адаптация ног под рельеф (ступеньки, кочки) без использования сложного IK.

---

## 2. Иерархия скелета (Bone Hierarchy)

Текущая система (только правая рука) расширяется до полноценного торса и ног. Базовой точкой привязки остается `Camera` (глаза).

### Дерево костей:
- **`cam_root`** (0, 0, 0 относительно камеры)
    - **`neck`** (смещена чуть назад на -0.1 по Z, чтобы избежать клиппинга камеры в грудь)
        - **`torso_upper`** (грудь)
            - **`shoulder_l`** -> `forearm_l` -> `hand_l`
            - **`shoulder_r`** -> `forearm_r` -> `hand_r`
            - **`spine`** (позвоночник, идет вниз)
                - **`pelvis`** (таз)
                    - **`thigh_l`** -> `calf_l` -> `foot_l`
                    - **`thigh_r`** -> `calf_r` -> `foot_r`

---

## 3. Механики анимации (Data-Driven FK)

Поскольку FABRIK IK временно не используется, все движения задаются через JSON-треки в `src/main/resources/zenith/animations/`.

### 3.1. Базовые состояния (Walk/Sprint)
В `walk.json` и `sprint.json` добавляются треки для костей ног.
- Взмахи бедра (`thigh_x`) и сгиб колена (`calf_x`).
- При ходьбе ноги должны поочередно появляться в нижней части экрана.

### 3.2. Паркур: Состояние HANGING (Висение)
В `climbing.json` (или новом `hanging.json`) прописывается:
- Ноги расслаблены, бедра согнуты под углом ~30-45 градусов.
- При взгляде вниз игрок видит свои болтающиеся ноги.
- Добавляется небольшое процедурное покачивание через `sine` в коде.

### 3.3. Паркур: Состояние CLIMBING (Подъем)
- Цикличная анимация: левая нога поднимается высоко (колено почти у камеры), затем правая.
- Создается визуальное ощущение, что персонаж отталкивается от стены.

---

## 4. Procedural Leg Placement (Упрощенный Foot IK)

Для реализации эффекта "ноги на ступеньках" используется аддитивный метод:

1.  **Raycast**: Каждый кадр из мировых координат колен (`thigh_l/r`) пускается луч вертикально вниз.
2.  **Distance**: Вычисляется расстояние до поверхности. 
3.  **Adjustment**: Если расстояние меньше длины ноги (препятствие):
    - В коде вычисляется `offsetAngle = currentDistance / legLength`.
    - Это значение **аддитивно прибавляется** к текущему углу поворота кости `thigh` из JSON.
4.  **Результат**: Нога "подтягивается" вверх, если под ней ступенька, и опускается, если там яма, сохраняя общую фазу ходьбы.

## 5. Техническая реализация

### 5.1. Модель (hands.json)
- Расширение массива `bones`.
- Добавление кубов для торса, бедер, голеней и ботинок.
- Настройка `uv` для использования новых частей текстуры.

### 5.2. Рендеринг (ViewmodelRenderer.java)
- Обновление весов для новых костей (для Mining Heat).
- Реализация `BodyController` — класса, который будет отвечать за поворот торса вслед за камерой с задержкой (Deadzone).

### 5.3. Шейдер (viewmodel_fragment.glsl)
- Возможное добавление градиентного затухания (Dithering/Alpha) в районе шеи, чтобы плавно "растворить" модель перед глазами и исключить случайное перекрытие обзора.

---

## 6. Этапы внедрения
1. **Этап 1**: Создание полной модели скелета в `hands.json` (визуальный тест торса и левой руки).
2. **Этап 2**: Добавление треков ног в `walk.json`.
3. **Этап 3**: Реализация `BodyController` для связи торса с вращением камеры.
4. **Этап 4**: Внедрение рейкастов для процедурного подъема ног.
5. **Этап 5**: Полировка паркур-анимаций (CLIMBING/HANGING).
