# План внедрения RPG-системы "Zenith" (Ultra Data-Driven)

Этот план описывает архитектуру, где все RPG-параметры вынесены в JSON для максимальной гибкости.

## 1. Реестры Определений (Registries)
Все новые сущности регистрируются через `DataLoader` из папки `resources/zenith/registry/`.

### 1.1 Характеристики (`stats/*.json`)
*   **Гибкость**: Новая характеристика добавляется созданием JSON-файла.
*   **Связь**: Код запрашивает стат по `Identifier`.
*   **Пример**: `mobility.json` определяет базовую скорость и влияние на анимации.

### 1.2 Редкость (`rarities/*.json`)
*   Определяет цвет (`color`), количество бонусов (`affix_slots`) и шанс появления в луте (`weight`).
*   Позволяет добавлять новые градации (например, "Relic" или "Cursed").

### 1.3 Аффиксы (`affixes/*.json`)
*   Наборы модификаторов статов.
*   Имеют теги совместимости (`applicable_to`), чтобы на куртку не выпал бонус "Острота".

## 2. Физические Лутбоксы (World Interaction)
Лутбокс — это не меню, это физический процесс.

*   **Data-Driven Блоки**: В JSON блока добавляется компонент `zenith:loot_container`.
*   **Параметры**:
    *   `loot_table`: Ссылка на таблицу лута.
    *   `interaction_time`: Базовое время вскрытия (уменьшается **Сноровкой**).
    *   `required_tool_tag`: Тег инструмента (например, `zenith:crowbar_tools`).

## 3. Таблицы Лута (`loot_tables/*.json`)
Гибкая система распределения предметов:
*   **Rolls**: Количество попыток.
*   **Pools**: Группы предметов с весами.
*   **Auto-Affix**: Флаг, заставляющий систему генерировать случайные статы для выпавшего предмета на основе его редкости.

## 4. Технические этапы реализации

### Шаг 1: Generic Registry & Stat Engine
*   Реализация `StatRegistry` и `RarityRegistry`.
*   Создание универсального `StatContainer` для игроков и предметов.
*   Обновление `ItemStack` для хранения динамических данных об аффиксах и редкости.

### Шаг 2: Loot Generation System
*   Парсер `LootTable`.
*   Алгоритм "Roll & Affix": выбор редкости -> выбор предмета -> выбор аффиксов -> создание `ItemStack`.

### Шаг 3: Тактильное Вскрытие
*   Интеграция в `InputManager`: при зажатии ПКМ на лутбоксе запускается таймер вскрытия.
*   Анимация: проигрывание анимации "взлома" или "рытья" (в зависимости от тега блока).

### Шаг 4: Динамический UI
*   `StatsComponentUI`: Компонент для `GUIRegistry`, отображающий список статов.
*   `TooltipBuilder`: Система сборки тултипа из "кусочков" (редкость, название, статы, описание), описанная в JSON.

---
**Принцип**: Код — это только рельсы. Весь контент — это JSON-вагоны.
