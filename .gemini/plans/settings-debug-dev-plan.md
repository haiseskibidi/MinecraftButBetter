# План реализации: Система настроек, Бинды и Экран отладки

Этот план описывает этапы создания комплексной системы настроек, перевода ввода на абстрактные экшены и добавления функционала для разработчиков (Debug Overlay).

## Этап 1: Ядро системы настроек (SettingsManager)
**Цель**: Создать централизованное хранилище настроек, сохраняющееся в `options.json`.

1. **Создать `SettingsManager.java`**:
   - Реализовать паттерн Singleton.
   - Поля: `fov`, `mouseSensitivity`, `devMode`, `keyBinds` (Map<String, Integer>).
   - Методы: `load()`, `save()`, `resetToDefaults()`.
   - Использовать GSON для сериализации.
2. **Интеграция в `GameLoop`**:
   - Вызов `SettingsManager.getInstance().load()` при инициализации движка.
   - Вызов `save()` при выходе из игры.

## Этап 2: Абстракция ввода (Refactoring Input)
**Цель**: Избавиться от хардкода клавиш в `InputManager` и `GameLoop`.

1. **Определить список Action ID**:
   - Движение: `move_forward`, `move_back`, `move_left`, `move_right`.
   - Действия: `jump`, `sprint`, `sneak`, `drop`, `inventory`, `journal`.
   - Системные: `pause`, `debug_menu` (F3), `editor_toggle` (F8).
2. **Модифицировать `InputManager.java`**:
   - Добавить методы `isActionPressed(String actionId)`.
   - Заменить все вызовы `window.isKeyPressed(GLFW_KEY_...)` на проверку через `SettingsManager`.
3. **Обновить `GameLoop.java`**:
   - Перевести открытие инвентаря, паузы и дневника на новую систему экшенов.

## Этап 3: Экран отладки (Debug Overlay)
**Цель**: Визуализация FPS и координат при включенном `devMode`.

1. **Создать `DebugRenderer.java`** (или добавить метод в `HUDRenderer`):
   - Отрисовка текста в правом верхнем углу.
   - Вывод: `FPS: [value]`, `XYZ: [x, y, z]`, `Chunk: [cx, cz]`.
2. **Логика отображения**:
   - Показывать оверлей только если `SettingsManager.getInstance().isDevMode()` возвращает `true`.
   - Добавить хоткей `F3` для быстрого переключения видимости (если нужно, либо оставить только через настройки).

## Этап 4: Интерфейс настроек (SettingsScreen)
**Цель**: Создать графический интерфейс для изменения параметров.

1. **Создать `SettingsScreen.java`**:
   - Наследовать `Screen`.
   - Реализовать горизонтальное меню вкладок (Tabs): `CONTROLS`, `VIDEO`, `DEVELOPER`.
2. **Реализовать компоненты управления**:
   - **Keybind Button**: При клике входит в состояние "Listening", записывает следующую нажатую клавишу.
   - **Slider**: Для FOV и чувствительности мыши.
   - **Toggle**: Для VSync и Dev Mode.
3. **Интеграция в `PauseScreen`**:
   - Кнопка "Настройки" должна открывать `SettingsScreen`.

## Этап 5: Панель разработчика (Live Inspector)
**Цель**: Реализовать прототип инспектора JSON-данных.

1. **Создать `DevInspectorScreen.java`**:
   - Древовидный список всех объектов в реестрах (Blocks, Items).
   - Панель свойств справа, генерируемая через Reflection.
2. **Hot-Reload Логика**:
   - Сохранение измененных значений обратно в файлы ресурсов.
   - Принудительный вызов `Renderer.rebuildAllChunks()` при изменении свойств блоков.

## План верификации
1. Проверить, что `options.json` создается и читается корректно.
2. Убедиться, что переназначение клавиши прыжка работает мгновенно.
3. Проверить, что Debug Overlay не мешает основному геймплею и правильно скрывается.
4. Убедиться, что FPS считается корректно и XYZ обновляются при движении.
