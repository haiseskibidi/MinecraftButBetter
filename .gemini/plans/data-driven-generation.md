# План: Data-Driven Генерация Мира (Zenith v2.0)

## Цель
Полностью избавиться от хардкода в пайплайне генерации ландшафта (плоский мир, модульная сетка города, деревья в коде, случайный разброс лута). Перевести генерацию на гибкие JSON-конфигурации, чтобы управлять рельефом, биомами, структурами и спавном без изменения Java-кода.

---

## Фаза 1: Data-Driven Биомы и Рельеф (Terrain Core)

Текущий `BiomeGenerator` содержит захардкоженные константы для 8 биомов. Текущий `CityLayoutStep` генерирует плоский мир на высоте 60.

1.  **Создание схемы `BiomeDefinition`**:
    *   Папка: `src/main/resources/zenith/generation/biomes/*.json`.
    *   Поля: `id`, `temperature` (диапазон), `humidity` (диапазон), `base_height`, `height_variation`, `surface_block`, `underground_block`, `features` (ссылки на пулы структур/деревьев), `scavenge_rules`.
2.  **BiomeRegistry & DataLoader**:
    *   Написать загрузчик биомов.
    *   Обновить `BiomeGenerator` для выборки загруженных биомов на основе 2D-шума (температура/влажность), вместо `enum`.
3.  **TerrainStep (Замена хардкода CITY_LEVEL)**:
    *   Написать шаг, который интерполирует `base_height` и генерирует 3D-рельеф (холмы, низины) с помощью комбинации шумов (Continentalness, Erosion).

## Фаза 2: Клеточные Города и Дороги (Voronoi/Cellular Cities)

Текущая сетка улиц генерируется примитивным делением по модулю `worldX % 30 == 0`, создавая бесконечный город.

1.  **CityGenerationSettings (JSON)**:
    *   Вынести настройки в `generation/cities.json`: `cell_size`, `road_width`, `city_density`, `asphalt_block`, `ruin_chance`.
2.  **Cellular Noise для Центров**:
    *   Использовать клеточный шум (Voronoi) для определения "ячеек" городов. Если плотность в центре ячейки высока, там генерируется городская зона.
3.  **Адаптивные дороги (Road Carving)**:
    *   Дороги должны соединять центры ячеек или строиться по графам.
    *   Дороги "срезают" рельеф (терраформирование), выравнивая блоки под собой, чтобы не было дорог под углом 90 градусов на холмах.

## Фаза 3: Структуры, Растительность и Повороты (Data-Driven Features)

Деревья сейчас генерируются через хардкод `generateMutatedTree`. Здания не поворачиваются к дороге.

1.  **Деревья как Префабы**:
    *   Удалить хардкод. Создать 5-10 структур (JSON) в `structures/trees/` (дуб, береза, мертвые деревья, пни).
    *   Биомы будут содержать пулы спавна: `"trees": [{ "id": "zenith:oak_tree", "weight": 100, "chance": 0.05 }]`.
2.  **Вращение Структур (Rotation Matrix)**:
    *   Реализовать поддержку поворотов (0, 90, 180, 270 градусов) в `StructureTemplate.build()`.
    *   При спавне зданий вдоль дорог алгоритм должен вычислять вектор нормали дороги и поворачивать фасад (определенный в префабе) к асфальту.
3.  **FeatureRegistry**:
    *   Универсальный менеджер спавна "Фичей" (деревья, валуны, кусты, здания), опирающийся на веса (Weights) из настроек биома/города.

## Фаза 4: Контекстный Спавн Сущностей и Лута (Contextual Spawning)

Палки, металлолом и враги спавнятся случайно по всей карте.

1.  **ScavengeRules (JSON)**:
    *   Переписать `ScavengeSettings`.
    *   Правила в JSON:
        *   `item`: `"zenith:scrap_metal"`
        *   `allowed_surfaces`: `["zenith:asphalt", "zenith:concrete"]`
        *   `chance`: `0.02`
        *   `biome_tags`: `["city", "industrial"]`
2.  **Entity Spawning Rules (JSON)**:
    *   Настроить спавн `ScoutEntity` и других мобов через JSON (в привязке к биому или структуре).
    *   Поддержка `SpawnContext`: "Только в темноте", "Только в радиусе 15 блоков от структуры 'ruined_house'".
3.  **ScavengeDecorationStep V2**:
    *   Алгоритм проверяет `surface_block` и запрашивает у `ScavengeRegistry` пул валидных предметов для этой поверхности в текущем биоме. (Например, палки падают только на траву, шанс возрастает возле блоков древесины).

## Порядок выполнения (Execution Order)

Рекомендуется двигаться строго по этому порядку, чтобы не сломать игру в процессе рефакторинга:

1.  **[ ] Шаг 1: Инфраструктура JSON (Registries).** Написать парсеры для `BiomeDefinition` и перевести деревья в формат структур (`.json` + `PrefabManager`).
2.  **[ ] Шаг 2: Вращение структур.** Внедрить математику поворотов в `StructureTemplate`, чтобы подготовить почву для умных городов.
3.  **[ ] Шаг 3: Terrain & Biomes.** Переписать `TerrainGenerator` и шаг Layout, внедрив холмы, биомные блоки земли/камня и загрузку из JSON. Города временно отключить.
4.  **[ ] Шаг 4: Умные города.** Реализовать Вороной-шум для городов, дороги, выравнивающие рельеф, и расстановку повернутых зданий фасадом к дороге.
5.  **[ ] Шаг 5: Контекстный лут.** Внедрить валидацию поверхностей (`allowed_surfaces`) в систему `ScavengeDecorationStep`.
