# Архитектурный проект: Zenith Physical Hands 2.0

Этот документ описывает полную переработку системы рук и анимаций для достижения максимального реализма при минимальных затратах на ручное анимирование.

## 1. Новая Иерархия Костей (Skeleton 2.0)

Каждая рука (L/R) состоит из следующих узлов:
- **`shoulder`**: Базовая кость. Отвечает за общую позицию руки.
- **`forearm`**: Предплечье. Основной рычаг.
- **`wrist`**: Запястье. Позволяет кисти "доворачивать" при инерции.
- **`palm`**: Ладонь. База для пальцев и сокетов.
- **`thumb`**: Большой палец (1 фаланга для упрощения).
- **`fingers`**: Группа остальных 4-х пальцев (1 узел). Позволяет сжимать кулак.

---

## 2. Система Динамических Сокетов (Sockets)

Внедряется универсальный механизм привязки предметов к костям.
- **Правило именования**: Любой узел, начинающийся с `socket_`, считается точкой крепления.
- **Базовые точки**:
    - `socket_palm_r`: Для инструментов.
    - `socket_wrist_l`: Для часов и гаджетов.
    - `socket_back_l`: Для наплечных или наручных сканеров.
- **Логика**: Предмет в JSON указывает `attachmentSocket`. Рендерер находит этот узел в скелете и умножает его `globalMatrix` на локальную матрицу предмета.

---

## 3. Слоистая Система Анимации (Layered Anim)

Итоговая трансформация кости = **Layer A (JSON)** + **Layer B (Procedural)** + **Layer C (Pose)**.

### Слой A: Традиционный JSON
- Задает только широкие движения: взмахи при ходьбе, удары.
- Работает только с `shoulder` и `forearm`.

### Слой B: Процедурная Физика (Springs)
- **Sway**: Задержка поворота кисти при вращении камеры.
- **Bobbing**: Процедурная синусоида ходьбы, зависящая от реальной скорости игрока.
- **Breathing**: Мягкое покачивание (idle) в покое.

### Слой C: Статические Позы (Grip Engine)
- Система переключает "форму" кисти.
- `FIST`: При ударе рукой.
- `RELAXED`: Когда руки пусты.
- `GRIP`: Когда в руке топор/лом.

---

## 4. Диегетический Интерфейс (Gadet Integration)

- **Часы Zenith**: Отдельный 3D-объект (или плоскость), привязанный к `socket_wrist_l`.
- **Рендеринг**: Использует отдельный шейдер или текстурный атлас, который обновляется динамически (например, выводит цифры HP/Noise).
