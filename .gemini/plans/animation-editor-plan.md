# План разработки: Animation Editor (Студия Анимации)

## 1. Цель
Создать специализированный режим (сцену) внутри игры для визуального редактирования скелетных анимаций (`Viewmodel`). Система должна позволять настраивать кости, создавать ключевые кадры, управлять интерполяцией и сохранять результат в JSON-формате, совместимом с текущим движком.

## 2. Архитектура системы

### 2.1. Базовый экран (`AnimationEditorScreen`)
- Реализация интерфейса `com.za.zenith.engine.graphics.ui.Screen`.
- Изолированный рендеринг: отдельный `Framebuffer` для 3D-вьюпорта и оверлей для UI.
- Состояние редактора: `ActiveAnimation`, `SelectedBone`, `CurrentTime`, `IsPlaying`.

### 2.2. Компоненты UI (на базе существующего `UIRenderer`)
- **Timeline (Таймлайн)**: Нижняя панель со шкалой времени (0.0 - 1.0 или секунды). Отображение ромбиков (ключевых кадров) для каждой дорожки.
- **Bone Hierarchy Explorer**: Список всех костей текущей модели (`shoulder_r`, `forearm_r`, `hand_r`, `item_attachment_r`).
- **Property Inspector**: Панель редактирования значений текущего ключа (X, Y, Z, Pitch, Yaw, Roll) и выбора типа сглаживания (`linear`, `quad_out`, `smootherstep`).
- **Transport Controls**: Кнопки Play/Pause, Stop, Loop, скорость воспроизведения.

## 3. 3D Взаимодействие (Gizmos & Raycasting)

### 3.1. Система Gizmos (Манипуляторы)
- **Translation Gizmo**: Три стрелки (RGB) для перемещения кости/предмета по осям.
- **Rotation Gizmo**: Три кольца для вращения (Pitch, Yaw, Roll).
- **Логика**: При клике на ось гизмо, фиксируется вектор движения мыши и преобразуется в изменение значения трансформации.

### 3.2. Raycasting
- Выбор костей кликом по их мешам в 3D пространстве.
- Выбор осей гизмо.

## 4. Интеграция с движком анимаций

### 4.1. Live Preview
- Редактор работает напрямую с объектами `AnimationProfile` и `AnimationTrack`.
- Любое изменение значения в UI мгновенно обновляет `Viewmodel` через `AnimationEvaluator`.

### 4.2. Скелетная привязка
- Возможность менять "родителя" для визуализации (например, смотреть анимацию с предметом в руке или без).

## 5. Persistence (Сохранение и загрузка)

### 5.1. JSON Serializer
- Конвертация структуры `AnimationProfile` в JSON, идентичный файлам в `resources/zenith/animations/`.
- Поддержка экспорта "на лету" (кнопка Save).

### 5.2. File Browser
- Возможность выбрать существующую анимацию для редактирования.

## 6. Этапы реализации (Milestones)

### Этап 1: Каркас и Рендеринг (MVP)
- [ ] Создание `AnimationEditorScreen`.
- [ ] Настройка статической камеры и освещения для вьюпорта.

- [ ] Базовая панель со списком костей.

### Этап 2: Таймлайн и Ключи
- [ ] Реализация шкалы времени и ползунка.
- [ ] Логика добавления/удаления ключевых кадров (`Keyframe`).
- [ ] Интерполяция значений между кадрами в реальном времени.

### Этап 3: Манипуляторы (Gizmos)
- [ ] Отрисовка 3D-стрелок и колец.
- [ ] Математика взаимодействия (Drag-and-drop осей).
- [ ] Raycasting для выбора объектов.

### Этап 4: Полировка и Экспорт
- [ ] Панель выбора Easing-функций.
- [ ] Система сохранения в файл через `JFileChooser` или внутренний браузер.
- [ ] Горячие клавиши (Space - Play, Ctrl+S - Save, G - Grab, R - Rotate).

## 7. Технические сложности
- **Matrix Math**: Правильный расчет мировых координат гизмо относительно иерархии костей (локальные vs мировые трансформации).
- **UI Complexity**: Создание интерактивного таймлайна на текущем простеньком `UIRenderer` потребует расширения системы компонентов.
