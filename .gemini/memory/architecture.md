# Архитектура проекта "MinecraftButBetter"

## Модульная структура
Проект разделен на несколько функциональных пакетов:

- **com.za.minecraft.engine**: Ядро движка.
  - **core**: Основной игровой цикл (GameLoop), управление окном (Window), таймер.
  - **graphics**: Система рендеринга (Renderer), работа с шейдерами, загрузка текстур, атласы.
  - **input**: Обработка ввода (InputManager). Здесь же находится логика расчета положения блоков и их метаданных.
  - **math**: Математические утилиты и векторы.
- **com.za.minecraft.world**: Система мира.
  - **blocks**: Определения блоков, их типов, форм (VoxelShape) и текстур.
  - **chunks**: Управление чанками (сетка 16x16x128).
  - **generation**: Генерация ландшафта (TerrainGenerator) на основе шума Перлина.
  - **physics**: Система коллизий и пересечений (AABB, VoxelShape, Raycast).
- **com.za.minecraft.entities**: Сущности.
  - **Player**: Основной класс игрока, наследующий физику и управляемый через InputManager.
- **com.za.minecraft.network**: Сетевой код.
  - Использует библиотеку Kryonet для обмена UDP/TCP пакетами.
  - Синхронизирует положение игроков и изменения в блоках мира.

## Механика блоков и метаданных
Для экономии ресурсов блоки в чанках представлены не объектами, а байтами. 
- Каждый блок имеет ID (тип) и Metadata (состояние).
- **Metadata** используется для:
  - Определения ориентации ступенек (взгляд игрока).
  - Определения типа полублока (верхний, нижний, северный, южный и т.д.).
- Логика рендеринга и физических столкновений считывает эти метаданные для корректного отображения и взаимодействия.

## Система рендеринга
- Использует OpenGL (через LWJGL 3).
- **Меши чанков**: Генерируются динамически, отрисовывая только видимые грани (Culling).
- **UI**: Кастомная система отрисовки 2D интерфейсов (хотбар, инвентарь).
