# Текущий статус проекта "MinecraftButBetter"

## Реализованные фичи
- **Survivor's Tablet (Дневник выжившего)**:
  - Полностью реализована модульная система обучения.
  - Современный карточный UI (50% ширины, 65% высоты экрана) с поддержкой прокрутки контента.
  - **ScrollPanel Component**: Универсальный UI-компонент с аппаратным отсечением (`glScissor`), решающий проблему "вылетающего текста".
  - **Dynamic Recipe Rendering**: Автоматическая отрисовка схем крафта прямо из игровых файлов.
    - Скалывание: отображение 5x5 сетки паттерна с циклической сменой допустимых материалов.
    - Пень: отрисовка ингредиентов, инструмента и счетчика ударов с текстовой тенью для читаемости.
  - Система категорий и статей полностью Data-Driven (JSON + `.index`).
- **Расширение In-World Crafting (Пень)**:
  - Добавлена поддержка нескольких ингредиентов (массив `ingredients`) в рецептах.
  - Реализован маппинг пустой руки на технический предмет `minecraft:hand`.
  - Добавлены/обновлены рецепты Tier 1: нити, заточенная палка, каменный топор, мешочек.
- **Гибкое скалывание (Napping)**:
  - Система переведена на поддержку списка входных материалов (`List<Identifier>`).
  - Все базовые инструменты (нож, топор, кирка, стрела) теперь можно делать как из кремня, так и из обычного камня.
- **Исправление системных ошибок**:
  - Исправлен баг "бессильной руки": теперь при пустой руке используется скорость 0.2, что позволяет ломать траву и медленно добывать землю/дерево.
  - Исправлен NPE в `StumpBlockDefinition` при клике пустой рукой.
- **Рефакторинг взаимодействия с миром (v2)**:
  - **Clean InputManager**: из `InputManager` удален хардкод логики для пней, ямного обжига, костров и генераторов. Логика перенесена в `onUse()` соответствующих `BlockDefinition`.
  - **Data-Driven Interactions**: введена поддержка типов блоков через `BlockTypeRegistry`. Новые типы: `campfire`, `pit_kiln`, `unfired_vessel`, `log`.
  - **Missing Assets Fallback**: введена система "шахматки" (пурпурный/черный) в `Texture.java`. При отсутствии ресурса игра не падает, а подставляет визуальный заглушку.
  - **Lazy Ticking Optimization**: внедрена система умного обновления `BlockEntity`. Сущности теперь могут сообщать `shouldTick() == false`, чтобы не нагружать `World.update`.
- **Универсальная Модульная Система Инвентаря (v3)**:
  - Внедрена полная абстракция данных через интерфейс `IInventory`.
  - **Data-Driven GUI**: все окна теперь описываются в JSON.
  - **CSS-like Layout System**: внедрена поддержка якорей (`anchor`) и гибкого выравнивания (`alignX`, `alignY`).
  - **Автоматическое Центрирование**: реализован расчет габаритов групп в `InventoryLayout` для идеального горизонтального позиционирования.
  - **Рефакторинг Инвентаря Игрока**:
    - Конфигурация: 9 (хотбар) + 8 (карманы 2x4) + 8 (снаряжение) + 8 (сумка) = 33 слота.
    - Исправлен баг `ArrayIndexOutOfBoundsException` (TOTAL_SIZE увеличен до 33).
    - Хотбар инвентаря идеально выровнен по горизонтали и поднят выше для предотвращения наложения на HUD.
  - **Developer Panel (Creative Menu) (v2.0)**:
    - Реализован плавный скроллинг через универсальный компонент `ScrollPanel`.
    - Добавлена поддержка `glScissor` для чистого отсечения иконок при прокрутке.
  - **Advanced Inventory Interactions (v3.2)**:
    - **Mouse Tweaks Style**: Реализован Shift-Drag для быстрого перемещения предметов и двойной клик для автоматического сбора стака.
    - **Hotbar Swapping**: Добавлена поддержка клавиш 1-9 для мгновенного перемещения предметов в хотбар.
    - **Data-Driven Stacking**: Внедрена поддержка `maxStackSize` в JSON предметов.
      - По умолчанию: блоки (128), инструменты (1), остальные (16).
      - Переопределено для: `pouch`, `fuel_canister`, `hand`, `fire_starter` (все по 1).
    - **Z-Order Fix**: Отрисовка предмета в руках и тултипов перенесена в самый конец, теперь они всегда поверх панелей.
    - **Return Logic**: Если кликнуть по тому же предмету в панели, который уже в руках, он "возвращается" (исчезает из курсора).
    - Исправлен баг с отсутствующими блоками (трава и др.) из-за лимита в 84 слота.
- **Универсальная Система Тинтинга (Universal Tinting)**:
  - Реализована поддержка тега `minecraft:tinted` в JSON блоков.
  - Шейдер `fragment.glsl` теперь использует знак `blockType` (отрицательный ID) для определения необходимости наложения тинта (трава, листва).
  - Удален хардкод UV-координат из шейдера и `Renderer.java`.
- **Stripped Logs (Обтёсанные бревна)**:
  - Добавлены обтёсанные варианты для всех типов деревьев (акация, береза, темный дуб, джунгли, ель).
  - Полная поддержка в `Blocks.java`, `.index` и JSON ресурсах.
- **Tier 1: Foraging & Scavenging**:
  - Реализована механика сбора 3D ресурсов с земли (палки, камни, кремень).
  - Внедрен `ResourceEntity` для рендеринга честных 3D-моделей предметов в мире через `ItemMeshGenerator`.
  - **Универсальный In-World Crafting API (v2)**:
    - Реализован интерфейс `ICraftingSurface` и `CraftingLayoutEngine`.
    - Поддержка адаптивной сетки 3x3 с фиксированными якорями.
    - Механика дискретных ударов (`isNewClick`).
    - Управление: ПКМ — положить, Shift+ПКМ — забрать, ЛКМ (с инструментом) — удар.
    - Система полностью Data-Driven.
    - Добавлена логика превращения бревна (`oak_log`) в рабочий пень (`stump`) при ударе топором.
  - Реализовано динамическое масштабирование (`visualScale`) предметов через JSON.
  - Реализован рейкаст по сущностям и мгновенный сбор ресурсов ЛКМ.
  - Настроена Data-Driven генерация ресурсов через `ScavengeDecorationStep` и `scavenge.json`.
  - **Система Хитбоксов**:
    - Реализована универсальная система генерации AABB на основе текстур (`TextureAABBGenerator`).
    - `ResourceEntity` теперь имеют идеально точные хитбоксы.
    - Физика коллизий (`Entity.move`) и выбор блоков (`Raycast`) теперь полностью учитывают `VoxelShape`.
  - Улучшен UX строительства: превью блоков теперь отображается только при зажатой клавише Shift.
  - Добавлены базовые блоки и предметы Tier 1: `unfired_vessel` и `fired_vessel` с кастомными хитбоксами.
  - Реализована честная система дропа: блоки выбрасывают `ItemEntity`.
  - Добавлена глина (`clay`) и комочки глины (`clay_ball`).
  - Реализована система вероятностного дропа: `dropChance` в JSON.
  - Добавлена трава (`short_grass`) и солома (`straw`).
  - **Высокая трава (Tall Grass & Double Plants)**:
    - Архитектурная поддержка через `PlacementType.DOUBLE_PLANT`.
    - Механика установки двух блоков и синхронного разрушения.
    - Умный рендеринг: поддержка `upperTexture`.
    - Настроена генерация в мире через `OvergrowthStep`.
  - **Исправление физики предметов (Weight Fix)**:
    - Исправлена ошибка ускорения легких предметов.
    - Внедрена универсальная формула затухания скорости (Clamp 0.1 - 0.98).
  - **Slot Ghost Icons (Placeholders) (NEW)**:
    - Внедрена система плейсхолдеров через JSON.
    - Реализован эффект Grayscale через фрагментный шейдер.
    - Поддержка полупрозрачных серых иконок для брони и сумок.
  - **Hybrid Strict Item Validation (NEW)**:
    - Реализован флаг `strict` в `EquipmentComponent`.
    - Мешочек нельзя положить в слот шлема, но можно носить в кармане или аксессуарах.
  - **Advanced Relative GUI Layout (v3.1)**:
    - Внедрена система двухпроходного лейаута (`InventoryLayout`).
    - Поддержка `relativeTo`, `relativeAlign`, `fixedOffsetX/Y`.
  - **Tier 1: Scavenging Expansion**:
    - Добавлена добыча растительных волокон (`plant_fiber`) из высокой травы.
    - [x] Плетение мешочка (`pouch`) из 4 ниток. Мешочек дает +8 слотов в инвентаре.
    - [x] Система автоматического выброса вещей при снятии сумки.
    - **Интерактивное обтёсывание бревен**: превращение бревна в пень с помощью каменного ножа.
      - 16-зонная система прогресса (`carvingMask` в `StumpBlockEntity`).
      - Универсальный шейдер для маскирования текстур для `CarvingRenderer`.
      - Полностью Data-Driven через новый тип рецептов `carving`.
      - **Исправление (v1.1)**: решена проблема "битых текстур" и "двойного клика".
  - **Treecapitator System (Realistic Felling)**:
    - [x] **BFS-based Felling**: Универсальный алгоритм поиска связных компонентов дерева (логи и листва) только выше и вокруг точки удара.
    - [x] **Metadata BIT_NATURAL**: Введен 8-й бит метаданных (0x80) для отличия сгенерированных миром блоков от поставленных игроком.
    - [x] **Universal Felling Stages**: Реализовано 4 технических блока (`felling_stage_1..4`) с уменьшенными хитбоксами (Thinning).
    - [x] **Data-Driven Wood Mapping**: Создан `WoodTypeRegistry` (загрузка из `wood_types.json`) для поддержки множества видов деревьев без дублирования JSON-файлов.
    - [x] **Dynamic Texture Mapping**: `BlockTextureMapper` автоматически подставляет текстуры для стадий на основе тега `minecraft:logs`, игнорируя флаги метаданных через маску `& 0x07`.
    - [x] **Drop Consolidation**: Все ресурсы спавнятся в одной куче у основания дерева.
  - **Flora & Generation Expansion**:
    - [x] **Universal Flora**: Добавлены определения блоков и листвы для березы, ели, акации, джунглей и темного дуба.
    - [x] **Clean Generation**: Из `TreeGenerator` и `OvergrowthStep` полностью удален хардкод `OAK_LOG`. Типы деревьев выбираются динамически из `WoodTypeRegistry`.
  - **Моддинг-движок (Стабилизация)**:
    - ID предметов Tier 1 изменены на 401-404 для предотвращения перезаписи.
- **Моддинг-движок (Core)**:
  - Полностью удален хардкод `BlockType` и `ItemType`.
  - Классы-холдеры `Blocks`, `Items`, `PrefabManager` с авто-инициализацией через Reflection.
  - Внедрен `StructureRegistry` и оптимизировано хранение чанков.
- **Индустрия и Энергия**:
  - Реализован интерфейс `IEnergyStorage`.
  - Добавлены Бензогенератор, Кабели, Лампа и Аккумулятор.
- **Ядро и Графика**:
  - Базовый игровой цикл, OpenGL рендеринг, View Model.
  - Локализация (I18n) и Unicode-шрифты.
