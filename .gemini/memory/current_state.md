## Текущий статус проекта "Zenith"

## Реализованные фичи

### 🚀 Архитектура и Движок (Zenith v2.5)
- **Data-Driven Asset Management**: 
  - Полный рефакторинг `DataLoader`. Загрузка ресурсов теперь делегирована специализированным классам (`ActionDataLoader`, `BlockDataLoader`, `ItemDataLoader` и т.д.).
  - Поддержка **Hot Reload**: Реализована система синхронизации `ItemStack` при обновлении определений предметов "на лету" через рефлексию.
  - Интерфейс `LiveReloadable`: Унифицированный механизм отслеживания путей к исходным JSON файлам для всех игровых данных.
- **Robust Rendering**:
  - **Transparency/Translucency Split**: Разделена логика куллинга (`transparent`) и отрисовки в прозрачном слое (`translucent`). Исправлен X-ray баг окантовок.
  - **Palette Corruption Guard**: Добавлена защита от повреждения палитры чанка в `Chunk.java`, автоматическое восстановление поврежденных индексов блоков.
  - **UTF-8 Console**: Принудительная установка кодировки UTF-8 для `System.out` для корректного отображения кириллицы в логах Windows.

### 🎮 Геймплей и Механики
- **Procedural Auto-Grip v2.1**:
  - Реализовано попиксельное сканирование толщины рукоятки предмета в `ItemMeshGenerator`.
  - Поле `gripWidth` в `Mesh` позволяет пальцам игрока корректно обхватывать как тонкие кирки, так и толстые блоки.
- **Стабилизация Mining & Treecapitator**:
  - Устранены критические точки в `ChunkMeshGenerator`, вызывавшие `ACCESS_VIOLATION` при массовом обновлении блоков.
  - В `MiningController` добавлена проверка на исчезновение блока (reset при срубании дерева), предотвращающая обращение к удаленным данным.
- **Contextual Debug HUD (F3)**:
  - Добавлены новые флаги состояния блоков (`Tr` для Translucent).
  - Обновлена легенда дебаг-панели.

### 🛠 UI и Инструментарий
- **JSON-Centric GUI**: Переработка системы HUD и меню на использование конфигурационных JSON файлов.
- **DevInspector v4.2**: Стабилизация позиционирования элементов и расширение возможностей инспекции игровых объектов.
