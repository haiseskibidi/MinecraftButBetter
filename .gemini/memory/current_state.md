# Текущий статус проекта "MinecraftButBetter"

## Реализованные фичи
- **Tier 1: Foraging (NEW)**:
  - Реализована механика сбора 3D ресурсов с земли (палки, камни, кремень).
  - Внедрен `ResourceEntity` для рендеринга честных 3D-моделей предметов в мире через `ItemMeshGenerator`.
  - Реализовано динамическое масштабирование (`visualScale`) предметов через JSON.
  - Реализован рейкаст по сущностям и мгновенный сбор ресурсов ЛКМ.
  - Настроена Data-Driven генерация ресурсов через `ScavengeDecorationStep` и `scavenge.json`.
  - **Система Хитбоксов (NEW)**:
    - Реализована универсальная система генерации AABB на основе текстур (`TextureAABBGenerator`).
    - `ResourceEntity` теперь имеют идеально точные хитбоксы, соответствующие их визуальной форме.
    - Физика коллизий (`Entity.move`) и выбор блоков (`Raycast`) теперь полностью учитывают `VoxelShape` (актуально для плит, ступенек и неполных блоков).
  - Исправлен баг, позволявший игроку двигаться во время процесса скалывания камня (napping).
  - Улучшен UX строительства: превью блоков (плит, ступенек и др.) теперь отображается только при зажатой клавише приседа (Shift).
- **Моддинг-движок (Финальная стабилизация)**:
  - Полностью удален хардкод `BlockType` и `ItemType`. Проект переведен на динамические реестры.
  - Реализованы классы-холдеры `Blocks`, `Items` и `PrefabManager` с автоматической инициализацией через **Reflection**. Теперь поля заполняются на основе JSON-данных при запуске.
  - Внедрен `StructureRegistry`: все здания (префабы) вынесены в JSON-файлы в `resources/minecraft/structures/`.
  - Устранена зависимость логики от числовых ID во всех системах: генерация мира, UI, инвентарь, инструменты и сетевой код теперь используют динамические ссылки.
- **Моддинг-движок (Этап 4: Оптимизация и Миграция)**:
  - Весь проект переведен на использование `int` для идентификаторов блоков и предметов, что снимает лимит в 256 типов.
  - Радикально оптимизировано хранение данных в `Chunk`: вместо 3D-массива объектов используется одномерный массив `int[] blockData` с упаковкой данных (Type + Metadata). Это значительно снизило потребление RAM и нагрузку на GC.
  - Сетевой протокол обновлен для поддержки передачи 32-битных ID блоков.
  - Обновлены генераторы мира и структур для работы с новыми типами данных.
- **Моддинг-движок (Этап 3: Расширяемость Логики)**:
  - Реализован `ItemTypeRegistry` для динамического создания инстансов предметов различных типов.
  - Внедрена система **Data Components** (аналог современных версий Minecraft). Предметы теперь хранят свойства в компонентах (`FoodComponent`, `ToolComponent`), что позволяет легко комбинировать поведение.
  - Реализована система **Registry Events**: моды могут подписываться на события `subscribeBlocks` и `subscribeItems` для регистрации своего контента в правильный момент.
  - Рефакторинг `DataLoader`: теперь поддерживает парсинг секции `components` из JSON и автоматическое связывание с объектами Java.
- **Моддинг-движок (Стабилизация)**:
  - Исправлен критический баг `NullPointerException` при инициализации мира, связанный с порядком регистрации блоков.
  - Внедрена гарантированная регистрация базового блока `AIR` перед загрузкой внешних ресурсов.
  - Добавлены защитные механизмы в `BlockRegistry` и `ItemRegistry`, предотвращающие возврат `null` (автоматический fallback на `AIR`).
  - Улучшена отказоустойчивость `DataLoader`: теперь ошибка в одном JSON-файле не прерывает загрузку всего реестра.
- **Моддинг-движок (Этап 2: Динамический Реестр)**:
  - Создан универсальный класс `Registry<T>` для унификации регистрации любого контента (блоков, предметов).
  - Реализована миграция данных: блоки и предметы теперь хранятся в индивидуальных JSON-файлах в папках `blocks/` и `items/`.
  - Внедрена система индексации `.index` для надежного сканирования ресурсов внутри JAR и в dev-среде.
  - Обновлен `DataLoader` для поддержки распределенной загрузки и возможности переопределения ресурсов модами.
- **Моддинг-движок (Этап 1: Фундамент)**:
  - Реализован класс `Identifier` (формат `namespace:path`) для строковой идентификации всех игровых ресурсов.
  - Внедрен `BlockTypeRegistry` для динамической регистрации фабрик блоков (плиты, лестницы, технические блоки).
  - Реализован `DataLoader` для загрузки описаний блоков и предметов из JSON-реестров (`blocks.json`, `items.json`).
  - Рефакторинг `BlockRegistry` и `ItemRegistry`: теперь поддерживают поиск по `Identifier` и автоматическое наполнение из внешних данных.
  - Успешная де-хардкодизация определений блоков и предметов из Java-кода в ресурсы.
- **Глубокая интеграция MCP**:
  - Подключены и настроены серверы `memory` (Knowledge Graph), `github` и `java-lsp` в глобальном конфиге.
  - Построен начальный архитектурный граф знаний проекта (сущности Core, Graphics, World, Entities, Energy).
  - Обновлены стандарты разработки (`GEMINI.md`) с упором на использование MCP и LLM-friendliness (подробные докстринги, явные связи в графе).
- **Улучшение UX инвентаря**:
  - Реализовано деление стеков правой кнопкой мыши (взять половину / положить один).
  - Добавлены горячие клавиши: `Q` (выбросить один), `Ctrl+Q` (выбросить стек) — работают как в игре, так и в инвентаре.
  - Реализовано выбрасывание предметов при клике вне окон инвентаря.
  - Автоматическое объединение стеков при клике левой кнопкой мыши.
  - **Drag-to-Distribute**: Раскладка предметов «мазком» (ЛКМ — поровну, ПКМ — по одному). Умное выделение слотов.
  - **Quick Sort (клавиша Z)**: Автоматическая сортировка и объединение стаков в инвентаре.
  - **Тултипы**: Добавлено отображение числового ID предмета (полезно для разработки рецептов).
- **Ядро и движок**: Базовый игровой цикл (GameLoop), управление окном через GLFW, рендеринг через OpenGL.
- **Мир**: Генерация бесконечного (в теории) мира по чанкам, сохранение изменений блоков в оперативной памяти.
- **Сущности**: Физика игрока (гравитация, коллизии, прыжки, режим полета).
- **Инструменты и блоки (UPDATED)**:

  - Полный набор стандартных блоков (камень, земля, трава, дерево и т.д.).

  - Механика "Вертикального режима" (клавиша R) для установки полублоков в вертикальном положении.

## Последние изменения (Исправления и Оптимизация)
- **Устранение критических утечек памяти и зависаний**: 
  - Исправлен бесконечный цикл каскадного обновления чанков при генерации лесов (`World.setBlock` с флагом `notify=false`).
  - Внедрено жесткое кэширование мешей предметов и блоков в `Renderer.java`, предотвращающее переполнение видеопамяти.
  - Отменен спавн тысяч `BlockEntity` для каждого дерева. Теперь `StumpBlockEntity` создается лениво, только при взаимодействии с блоком (превращение WOOD в STUMP).
- **Исправлена плавность игры**: Устранены рывки камеры при открытии инвентаря (синхронизация перенесена в конец физического шага).
- **Удалены фризы**: Избавление от блокирующих циклов `while` при нажатии клавиш ESC и E. Теперь ввод неблокирующий.
- **Исправление Рендеринга и CTM**:
  - Исправлен критический баг в `Renderer.java`, где в шейдер передавались неверные UV-координаты (использовался `minV` вместо `maxV`). Это исправляет "бежевое стекло", поломку Connected Textures и отсутствие окрашивания травы/листвы.
  - Оптимизирован `ChunkMeshGenerator.java`: возвращено исключение отрисовки внутренних граней для стекла (как в стабильной версии), что снижает количество вершин без потери CTM-эффектов.
- **Исправление UI и Голода**:
  - Исправлен баг отображения количества предметов в хотбаре.
  - Исправлен критический баг отрисовки голода: при значении 0 больше не выводится `%.1f` вместо числа.
- **Исправление Генератора**:
  - Добавлена информация о состоянии (топливо, энергия) при взаимодействии с генератором.
