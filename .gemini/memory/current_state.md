# Текущий статус проекта "MinecraftButBetter"

## Реализованные фичи
- **Универсальная Модульная Система Инвентаря (v3) (NEW)**:
  - Внедрена полная абстракция данных через интерфейс `IInventory`. Поддерживаются игроки, сундуки и виртуальные реестры (`RegistryInventory`).
  - **Data-Driven GUI**: все окна теперь описываются в JSON (`player_inventory.json`).
  - **CSS-like Layout System**: внедрена поддержка якорей (`anchor`) и гибкого выравнивания (`alignX`, `alignY`).
  - **Автоматическое Центрирование**: реализован расчет габаритов групп в `InventoryLayout` для идеального горизонтального позиционирования.
  - **Рефакторинг Инвентаря Игрока**:
    - Конфигурация: 9 (хотбар) + 8 (карманы 2x4) + 8 (снаряжение) + 8 (сумка) = 33 слота.
    - Исправлен баг `ArrayIndexOutOfBoundsException` (TOTAL_SIZE увеличен до 33).
    - Хотбар инвентаря идеально выровнен по горизонтали и поднят выше для предотвращения наложения на HUD.
  - **Developer Panel**: перенесена в правый верхний угол, добавлена поддержка скроллинга и мгновенного обновления при смене режима (F3).
- **Tier 1: Foraging & Scavenging**:
  - Реализована механика сбора 3D ресурсов с земли (палки, камни, кремень).
  - Внедрен `ResourceEntity` для рендеринга честных 3D-моделей предметов в мире через `ItemMeshGenerator`.
  - **Универсальный In-World Crafting API (v2)**:
    - Реализован интерфейс `ICraftingSurface` и `CraftingLayoutEngine`.
    - Поддержка адаптивной сетки 3x3 с фиксированными якорями (устраняет наложение предметов).
    - Механика дискретных ударов (`isNewClick`): прогресс крафта засчитывается только при физическом клике ЛКМ.
    - Управление: ПКМ — положить, Shift+ПКМ — забрать, ЛКМ (с инструментом) — удар.
    - Система полностью Data-Driven через рецепты типа `in_world_crafting`.
    - Добавлена логика превращения срубленного бревна (`oak_log`) в рабочий пень (`stump`) при клике топором.
  - Реализовано динамическое масштабирование (`visualScale`) предметов через JSON.
  - Реализован рейкаст по сущностям и мгновенный сбор ресурсов ЛКМ.
  - Настроена Data-Driven генерация ресурсов через `ScavengeDecorationStep` и `scavenge.json`.
  - **Система Хитбоксов**:
    - Реализована универсальная система генерации AABB на основе текстур (`TextureAABBGenerator`).
    - `ResourceEntity` теперь имеют идеально точные хитбоксы, соответствующие их визуальной форме.
    - Физика коллизий (`Entity.move`) и выбор блоков (`Raycast`) теперь полностью учитывают `VoxelShape` (актуально для плит, ступенек и неполных блоков).
  - Исправлен баг, позволявший игроку двигаться во время процесса скалывания камня (napping).
  - Улучшен UX строительства: превью блоков (плит, ступенек и др.) теперь отображается только при зажатой клавише приседа (Shift).
  - Добавлены базовые блоки и предметы для системы Tier 1: `unfired_vessel` (сырой кувшин) и `fired_vessel` (обожженный горшок). Для них настроены компактные кастомные хитбоксы через JSON.
  - Реализована честная система дропа: блоки теперь выбрасывают предметы как сущности (`ItemEntity`) на землю, а не добавляют в инвентарь напрямую.
  - Добавлена глина (`clay`) и комочки глины (`clay_ball`). Настроен выпад комочков из блока глины через JSON-поле `dropItem`.
  - Реализована система вероятностного дропа: в JSON блока теперь можно указать `dropChance` (от 0.0 до 1.0).
  - Добавлена трава (`short_grass`) и солома (`straw`). Из травы с шансом 20% выпадает солома (топливо для Tier 1).
  - **Высокая трава (Tall Grass & Double Plants)**:
    - Реализована архитектурная поддержка двойных растений через `PlacementType.DOUBLE_PLANT`.
    - Механика установки: при размещении одного предмета автоматически ставятся два блока (низ и верх) с разными метаданными (0 и 1).
    - Синхронное разрушение: при ломании любой части растения вторая удаляется автоматически.
    - Умный рендеринг: поддержка `upperTexture` в JSON позволяет использовать разные текстуры для верхней и нижней частей.
    - Исправлен рендеринг высокой травы в руке и инвентаре (отображается плоскостями, а не кубом).
    - Настроена генерация высокой травы в мире через `OvergrowthStep`.
  - **Исправление физики предметов (Weight Fix)**:
    - Полностью пересмотрена формула трения в `ItemEntity.java`.
    - Устранена ошибка ускорения ("ракетного эффекта") для легких предметов (вес < 0.7).
    - Внедрена универсальная формула затухания скорости, зависящая от веса, с безопасными лимитами (Clamp 0.1 - 0.98).
  - **Slot Ghost Icons (Placeholders)**:
    - Внедрена система плейсхолдеров для пустых слотов через JSON (`placeholders` в `GroupConfig`).
    - Поддержка полупрозрачных серых иконок (масштаб 0.7x) для брони, щитов и сумок.
  - **Strict Item-to-Slot Validation**:
    - Реализован флаг `strict` в `EquipmentComponent`.
    - Предметы с этим флагом могут быть помещены **только** в слоты соответствующего типа (например, мешочек теперь нельзя положить в хотбар или карманы).
  - **Advanced Relative GUI Layout (v3.1)**:
    - Внедрена система двухпроходного лейаута (`InventoryLayout`).
    - Поддержка `relativeTo` и `relativeAlign` для привязки групп друг к другу.
    - Введены `fixedOffsetX/Y` для пиксельной точности (например, отступ 20px от хотбара для идеального центрирования 4 над 9).
    - Раздельное выравнивание осей через `relativeAlignX/Y`.
  - **Tier 1: Scavenging Expansion**:
    - Добавлена добыча растительных волокон (`plant_fiber`) из высокой травы.
    - Плетение мешочка (`pouch`) из 4 ниток через 5 ударов заточённой палкой на пне (In-World Crafting v2).
    - Мешочек функционирует как первое переносное хранилище (8 слотов).
    - Реализовано автоматическое выбрасывание предметов при снятии мешочка и мгновенное обновление GUI.
  - **Моддинг-движок (Стабилизация)**:
    - ID предметов Tier 1 (`straw`, `clay_ball`, `unfired_vessel`, `fired_vessel`) изменены на 401-404.
    - Это предотвращает их перезапись автоматическими `BlockItem`, которые создаются для каждого блока (включая блоки с ID 121-124).
- **Моддинг-движок (Core)**:
  - Полностью удален хардкод `BlockType` и `ItemType`. Проект переведен на динамические реестры.
  - Реализованы классы-холдеры `Blocks`, `Items` и `PrefabManager` с автоматической инициализацией через **Reflection**.
  - Внедрен `StructureRegistry`: все здания (префабы) вынесены в JSON-файлы.
  - Оптимизировано хранение чанков: использование `int[] blockData` для поддержки 32-битных ID.
- **Индустрия и Энергия**:
  - Реализован интерфейс `IEnergyStorage`.
  - Добавлены Бензогенератор, Электросети (кабели), Лампа и Аккумулятор.
- **Ядро и Графика**:
  - Базовый игровой цикл, OpenGL рендеринг, View Model (руки и предметы).
  - Локализация (I18n) и Unicode-шрифты.
